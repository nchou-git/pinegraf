# Sources

Pinegraf stores claims with source identifiers instead of anonymous facts. This
document describes the current source families, their trust level, and their
freshness model.

## Source Families

| Source | Trust | Freshness | Writes |
| --- | --- | --- | --- |
| `alumni_xlsx_v2` | High for spreadsheet fields, unknown recency | Snapshot date unknown; `as_of_date` and `last_verified_at` are null | `entity_attributes`, `alumni_profiles`, aliases |
| `raw_pages.id` / public URL | Medium; depends on source page quality | `raw_pages.fetched_at` records retrieval time | `facts`, `projects`, `connections`, `entity_attributes` |
| `wikidata:<qid>` | Medium-high for public structured facts | Verified when enrichment runs; `last_verified_at=now` | `entity_attributes` |
| Inference `source_ids` | Derived; confidence depends on underlying rows | Refreshed when reconciliation runs | inferred `connections`, `entity_consolidated` |
| Operational tables | Internal only | Current as of write time | `llm_usage`, `audit_events`, `audit_runs`, `extraction_cache` |

## Alumni Spreadsheet

The seed workbook is `data/alum_data.xlsx`, imported by:

```bash
python -m scripts.import_alumni_xlsx
```

Each spreadsheet row resolves or creates one entity using the resolver. The
importer writes these attributes when present:

- `class_year`
- `internship_company`
- `internship_location`
- `current_employer`
- `current_employer_website`
- `current_location`
- `eship_notes`

Rows are idempotent for `source='alumni_xlsx_v2'`. The workbook does not carry a
known snapshot date, so `as_of_date` and `last_verified_at` are null.

## Crawled Public Pages

The crawler stores source snapshots in `raw_pages`:

- `source_url` identifies the public page.
- `fetched_at`, `http_etag`, `http_last_modified`, and `content_sha256` describe
  the retrieval.
- `raw_html_gz` preserves compressed HTML when available.
- `page_text` is regenerated by `scripts.rebuild_page_text` after boilerplate
  modeling.

Parser-created claims should point to the raw page with `source_raw_page_id` and
carry `text_evidence` when the claim came from chunk extraction.

## LLM Extraction

Extraction is source-grounded but model-generated. Current controls:

- `chunk_page` records chunk boundaries and `page_chunks`.
- Triage and extraction responses are cached in `extraction_cache`.
- Every OpenAI call writes `llm_usage` with model, token counts, purpose, and
  dollar estimate.
- Full extraction emits `people`, `organizations`, `relationships`, and
  `projects` with `confidence` and inline `text_evidence`.
- `EXTRACTION_TIER_MODE` controls `mini_only`, `cascade`, or `frontier_only`.

Use confidence and validation verdicts when deciding whether a row should inform
analysis.

## Wikidata

Wikidata enrichment is run by:

```bash
python -m scripts.enrich_wikidata --limit 100
```

The source queries by entity label and aliases, then writes:

- `date_of_birth`
- `occupation`
- `current_employer`
- `education`
- `notable_for`

Rows use `source='wikidata:<qid>'`, `source_url` set to the Wikidata entity URL,
and `last_verified_at` set to the enrichment run time. The script throttles
requests to 10 per second and is idempotent by replacing the existing rows for
the same entity and QID.

## Derived Sources

`scripts.reconcile_entities` computes two derived outputs:

- `entity_consolidated` chooses profile fields by source date, source priority,
  agreement count, and confidence.
- Inferred `connections` use `source_ids` pointing to the facts, projects, or
  attributes that caused the rule to fire.

Derived rows are not independent evidence. Always inspect `source_ids` and
`derivation` for analyst-facing claims.

## Trust Guidance

- Prefer consolidated fields for quick display.
- Prefer raw `entity_attributes`, `facts`, `projects`, and explicit
  `connections` when auditing provenance.
- Treat inferred edges as hypotheses unless the underlying sources are strong.
- Treat public web pages and Wikidata as refreshable sources; rerun crawl,
  parse, enrichment, and reconciliation before high-stakes analysis.
